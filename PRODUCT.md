# 数字人格系统 — 产品方案

## 一句话定位

> 把每个人的沟通风格、知识、任务状态蒸馏成 AI Skill，
> 让 AI 成为你在飞书里的第二个自己，让员工经验成为企业永久资产。

---

## 核心功能

### 1. 风格蒸馏
从你的历史聊天记录 + 文档中提取：
- 常用句式 / 语气 / 表达习惯
- 对不同人（上级 / 同级 / 下级）的语气差异
- 常见话题的立场与态度
- 回复长度、标点、表情偏好

### 2. 回复副驾驶
别人发消息 → AI 先用你的风格生成草稿 → 弹窗给你 → 你一键发送或修改

```
消息来了
  ↓
AI 读取：消息内容 + 对话历史 + 你的任务状态
  ↓
生成 3 个候选回复（风格：你的）
  ↓
Bot 私信你："建议回复这个，点击发送"
  ↓
你确认 or 修改
```

### 3. 会议前简报
开会前 5 分钟自动推送：
- 这个会议的参与者是谁
- 你们过去聊过什么、上次未解决的问题
- 对方的沟通偏好
- 你答应过对方但还没做的事

### 4. 离线托管
你在开会 / 专注 / 休息时：
- 紧急消息 → 打扰你
- 普通消息 → 用你的风格回复"稍后跟进"
- 可自动处理的（问时间、问状态）→ 直接回
- 你回来看摘要："2小时内 4 条消息，1 条需要你亲自处理"

### 5. 难开口消息起草
拒绝、催款、提意见、说坏消息 → 最难写的消息
AI 用你的风格起草，给 3 个力度版本（委婉 / 直接 / 强硬）

### 6. 关系维护提醒
- 检测你多久没联系某人
- 关键合作方沉默提醒
- 重要节点（生日、项目周年）主动提示

## 企业级价值

### 痛点一：关键人员离职，知识全带走
**解法：** 员工的沟通风格、客户关系、处理习惯全部沉淀为 Skill
→ 新人接手无缝过渡，客户感知不到变化

### 痛点二：高管时间是瓶颈
**解法：** 助理用高管数字人格处理 80% 日常消息
→ 高管只看真正需要决策的 20%

### 痛点三：对外沟通风格不统一
**解法：** 统一企业对外人格，全公司回复风格一致
→ 客户体验显著提升

### 痛点四：新员工上手慢
**解法：** 新人写消息前，AI 先用"公司风格"润色建议
→ 3天上手公司沟通文化，不用3个月

## 技术架构

### 数据采集
```
lark-cli im +chat-messages-list   → 历史消息
lark-cli docs +fetch              → 历史文档
lark-cli task                     → 当前任务状态
lark-cli calendar                 → 日程信息
```

### 风格蒸馏
```
Claude API
  → 提取语气 / 句式 / 偏好
  → 生成结构化 Skill JSON
  → 存入飞书多维表格（base）
```

### 实时推荐
```
飞书 Webhook → 新消息触发
  → 读取 Skill（数字人格）
  → 读取任务状态
  → Claude 生成候选回复
  → im +messages-send 私信给用户
```

### 技术栈
- 后端：Node.js / Python
- AI：Claude API（claude-sonnet-4-6）
- 飞书集成：lark-cli + 飞书开放平台
- 存储：飞书多维表格（Base）
