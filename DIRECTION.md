# 飞书会议 Agent — 会议贡献量化 + 绩效分析

## 一句话定位

> 用 AI Agent 量化每个人在飞书会议中的真实贡献，
> 让会议不再"混时间"，让绩效分配有据可依。

---

## 痛点

1. **会议黑洞** — 开了 2 小时会，80% 时间只有 20% 人在发言，其余人在摸鱼
2. **贡献隐形** — 提了关键建议的人 vs 只是"嗯嗯"的人，在绩效上没有区分
3. **绩效主观** — 管理者凭感觉打分，缺乏客观数据支撑
4. **会议膨胀** — 没有人量化会议 ROI，无效会议越来越多

---

## 核心功能

### 1. 会议实时分析
飞书会议进行中，Agent 实时：
- 识别谁在发言、发言时长、发言次数
- 分析发言内容的质量（提出方案 vs 复述废话 vs 提出质疑）
- 标记关键决策时刻和谁推动了决策

### 2. 会议贡献值打分
会议结束后自动生成：
```
┌─────────────────────────────────────────┐
│  会议贡献报告                            │
│                                         │
│  总时长：60min                           │
│  参与者：5人                             │
│                                         │
│  贡献排名：                              │
│  🥇 张三  92分  提出核心方案+解决3个问题  │
│  🥈 李四  78分  关键质疑推动方案优化      │
│  🥉 王五  65分  补充数据支持              │
│  4️⃣  赵六  30分  发言少但精准             │
│  5️⃣  钱七  12分  基本沉默                 │
│                                         │
│  会议效率：72/100                        │
│  关键决策：3个                           │
│  待办事项：5个（已自动创建飞书任务）      │
└─────────────────────────────────────────┘
```

### 3. 自动化任务检查
- 从会议中自动提取 Action Items
- 创建飞书任务并分配责任人
- 跟踪任务完成情况
- 下次会议前自动汇报完成率

### 4. 绩效数据看板
- 个人：我的会议贡献趋势、擅长领域、改进方向
- 管理者：团队会议效率、谁在推动项目、谁在拖后腿
- 公司：部门间会议文化对比、会议 ROI 分析

### 5. 智能建议
- "这个会议其实可以不用开，发个文档就行" → 会议瘦身
- "张三在过去 5 个会议贡献持续下降，可能需要关注"
- "建议下次会议邀请李四，他在这个领域贡献度最高"

## 技术架构

```
飞书会议（音视频/屏幕共享）
         ↓
飞书实时字幕 API（语音转文字）
         ↓
Claude API（内容分析 Agent）
  ├── 发言质量分析
  ├── 贡献度打分
  ├── Action Item 提取
  └── 决策点标记
         ↓
飞书多维表格（数据存储）
         ↓
飞书 Bot（报告推送 + 任务创建）
         ↓
Dashboard（绩效看板）
```

### 关键 API

| 功能 | 飞书 API | 用途 |
|------|---------|------|
| 会议数据 | 飞书会议 API | 获取参与者、时长、录音 |
| 语音转文字 | 飞书实时字幕 / ASR | 会议内容转写 |
| 内容分析 | Claude API | 发言质量分析、贡献打分 |
| 数据存储 | 飞书多维表格 (Base) | 存储分析结果 |
| 消息推送 | 飞书 Bot / IM API | 推送报告和提醒 |
| 任务管理 | 飞书任务 API | 自动创建 Action Items |
| 日历 | 飞书日历 API | 会议前自动提醒 |

### 技术栈

- **后端：** Node.js / TypeScript
- **AI：** Claude API (claude-sonnet-4-6)
- **飞书集成：** 飞书开放平台 API + 飞书 CLI
- **存储：** 飞书多维表格 (Base)
- **前端：** 飞书妙搭 / Next.js Dashboard

## 差异化

| 维度 | 市面产品 | 我们 |
|------|---------|------|
| 飞书会议纪要 | 只做转写和摘要 | 量化每个人的贡献 |
| 绩效管理工具 | 靠上级主观打分 | 数据驱动，会议表现可量化 |
| 会议效率工具 | 统计时长/次数 | 分析内容质量和决策推动力 |

**最大差异化：不是帮你记会议内容，而是帮你评价谁在会议中真的有价值。**

---

## 风险与应对

| 风险 | 应对 |
|------|------|
| "会被网暴" — 量化贡献可能引发争议 | 提供"仅管理者可见"模式；强调是辅助工具不是审判工具 |
| 语音转写准确度 | 飞书自带 ASR，准确度有保障 |
| 会议隐私 | 仅分析已授权会议，支持 opt-out |
| 技术难度中等 | 2 周可交付 MVP |
