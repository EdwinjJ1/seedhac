# docIterate — 文档持续迭代

用于已有需求文档之后的增量更新。它关注新信息与旧文档之间的差异，把补充、删除、调整和新约束写回文档。

示例：

- “需求变更一下，登录先不做手机号”
- “补充一下验收标准”
- “把这个调整写进需求文档”

不要用于第一次创建需求文档；首次整理交给 requirementDoc。不要用于会议纪要或演示稿生成。
